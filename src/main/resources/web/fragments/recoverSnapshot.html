A snapshot called "CoherenceDemoSnapshot" has been recovered into the current cluster. This operation abandons all existing partitions for the service and loads data saved using the "Create Snapshot" command.

When you issue a snapshot operation against a service, the service is automatically suspended before the operation and resumed after the operation. This is to ensure the service and data is in a consistent state.

Note: If you have Federation enabled, or enable Federation after recovering a snapshot, you must issue a "Replicate All", to ensure both clusters are in sync..

There are a number of ways you can issue Persistence operations against services.

  1. Coherence Query Language (CohQL) - CohQL has commands for all Persistence operations. See the Coherence Documentation for more information.
  2. Coherence JVisualVM Plug-in - The plug-in has a "Persistence" tab, which allows you to issue all Persistence operations. See the Administration Guide for more information.
  3. WLST - When using Coherence in a WebLogic Server environment, WLST can be used to issue Persistence commands when connected to the Admin server. See the WebLogic Server Documentation for more information.
  4. Via a JMX tool such as JVisualVM or JConsole - Operations can be invokved on the PersistenceCoordinator MBean directly. See Coherence Documentation for more information.
  5. Using Java - See the examples shipped with Coherence for examples on how to do this.
